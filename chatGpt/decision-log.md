# Decision Log

## Purpose

This file stores durable decisions made in the workspace.

A decision belongs here if it is likely to matter again.

This is not a brainstorming file.
This is not a raw notes file.
This is a compressed record of meaningful choices and why they were made.

---

## Entry Format

## Decision Title
**Date:** YYYY-MM-DD  
**Area:** Architecture / Prompting / Workflow / Repo / Product / Agent System / Model Strategy / Other  
**Status:** Proposed / Accepted / Rejected / Revisited  
**Decision Owner:** User / Shared / AI-recommended-user-approved

### Context
What situation created this decision?

### Options Considered
- Option A
- Option B
- Option C

### Decision
What was chosen?

### Why
Why was this the best choice?

### Tradeoffs
What is lost, deferred, or accepted by making this choice?

### Reversibility
Easy / Medium / Hard

### Consequences
What now becomes true because of this choice?

### Next Actions
- [ ] Action 1
- [ ] Action 2
- [ ] Action 3

### Related Files / Chats
- `agent-backlog.md`
- `agent-patterns.md`
- relevant chat title
- repo area affected

## Example Entry

## Use Planner -> Worker -> Teacher -> User as the default workspace flow
**Date:** 2026-04-04  
**Area:** Workflow  
**Status:** Accepted  
**Decision Owner:** Shared

### Context
The workspace is being used to improve the Pariss project over time, sharpen prompting ability, and make ChatGPT more strategically useful.

### Options Considered
- Use ad hoc prompting
- Use Planner -> Worker only
- Use Planner -> Worker -> Teacher -> User

### Decision
Use Planner -> Worker -> Teacher -> User as the default workflow for substantial tasks in this workspace.

### Why
This produces stronger structure, better outputs, and durable learning. It also makes prompt quality itself an object of improvement.

### Tradeoffs
Slightly slower than direct prompting for trivial tasks. May be unnecessary for very small one-off questions.

### Reversibility
Easy

### Consequences
The workspace should favor prompts, templates, and files that support this flow.

### Next Actions
- [ ] Add this workflow to `workspace-os.md`
- [ ] Add reusable skeletons to `agent-patterns.md`
- [ ] Log critique patterns in `agent-backlog.md`

### Related Files / Chats
- `workspace-os.md`
- `agent-patterns.md`

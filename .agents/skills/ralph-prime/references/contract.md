# Planning contract

A decision-complete plan defines the problem, current evidence, novelty, scope, exclusions, dependencies, options and tradeoffs, selected approach, first vertical slice, deferred work, acceptance criteria, validation, risk, rollback, and next owner. Missing decisions remain explicit blockers.

Critique must cite target documents, tests, code, issues, or reproducible observations. Self-critique without evidence cannot change the lane. If a validation gate failed, the packet names the workflow, failing step, evidence excerpt, root-cause class, and one recovery route.

Durable learning records reusable rules and rejected approaches, but never transient scratch notes, private content, or a claim that the plan was implemented.

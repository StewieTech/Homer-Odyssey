# Benchmark Pattern Library

Use this file as the only baseline set for /improve-ui-ux-super. Compare interaction patterns, not brand identity.

## Onboarding and first value

- **ChatGPT**
  - High-signal patterns: composition primacy, immediate state clarity, predictable affordances.
  - Caution: avoid copying tone/brand; borrow sequence clarity only.

- **Duolingo**
  - High-signal patterns: fast lesson start, short correction loop, progress confidence cues.
  - Caution: avoid gamification density unless it improves emotional safety and retention.

- **Headspace**
  - High-signal patterns: calm first impression, low-friction setup, trust cues.
  - Caution: avoid decorative motion that slows learning tasks.

## Social and discovery flow patterning

- **Pinterest**
  - High-signal patterns: fast content scanning, visual hierarchy, easy recovery.

- **Hinge**
  - High-signal patterns: emotionally safe prompts, profile clarity, intentional prompts-to-action chain.
  - Caution: do not import dating-specific assumptions.

- **Airbnb**
  - High-signal patterns: trust cues, account/settings clarity, confidence in next action.

## Practice and progress systems

- **Notion**
  - High-signal patterns: progressive complexity reveal, clear section ownership, predictable navigation.

- **Spotify**
  - High-signal patterns: retention loops with short return signals and clear saved progress.

## Product direction rule

- Calibrate only on matching interaction pattern.
- If your product already outperforms a pattern in a dimension, log “keep current”.
- If a pattern is inapplicable, cite why clearly.
- Always cap issue generation at 3 findings per run.

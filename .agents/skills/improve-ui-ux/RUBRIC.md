# UI/UX Review Rubric

Use this rubric to make reviews consistent. Score only when scoring helps the user compare options or prioritize work; otherwise use it as a checklist.

## Scale

1. Broken or actively harmful
2. Usable with obvious friction
3. Functional but generic or under-resolved
4. Strong, clear, and trustworthy
5. Excellent, distinctive, and hard to simplify further

## Criteria

**User goal clarity**
- The screen has a clear reason to exist.
- The primary user goal is visible without explanation.
- Success is obvious to the user.

**First impression**
- The first 3 seconds communicate purpose, state, and next action.
- The screen feels intentional rather than assembled.
- The experience matches the user's emotional context.

**Action clarity**
- The primary action is visually and semantically dominant.
- Secondary actions are available without competing.
- Labels describe outcomes, not implementation.

**Interaction model**
- The user can act on the object of attention directly where possible.
- Controls are close to the thing they affect.
- The user can recover, retry, undo, or revise.

**Cognitive load**
- Choices are staged.
- Context stays visible where it is needed.
- The screen uses recognition over recall.

**Feedback and state**
- Actions receive immediate feedback.
- Loading, empty, success, error, disabled, and in-progress states are designed.
- State changes are specific enough to build trust.

**Learning loop**
- The user can attempt, receive feedback, retry, and see progress.
- Corrections are timely and emotionally safe.
- The assistant or feedback system feels context-aware rather than generic.

**Premium feel**
- Hierarchy, spacing, type, motion, copy, and contrast feel deliberate.
- The interface feels calm, fast, intelligent, and trustworthy.
- Polish supports the task instead of distracting from it.

**Accessibility**
- Keyboard, screen reader, contrast, focus, touch target, readable size, and reduced-motion needs are considered.
- Accessibility is treated as core usability, not a final pass.

**Benchmark calibration (optional for high-impact reviews)**
- Compare with 5-10 relevant best-in-class apps chosen by interaction pattern, not popularity alone.
- Rank the current design or options on first 3-second clarity, interaction directness, emotional safety, state polish, premium feel, and time to first value.
- Call out where your product should follow conventions and where it should intentionally diverge.

## Output Format

When producing a review, use this order:

1. Strongest decision point and confidence percentage
2. Overall score and one-sentence diagnosis
3. Must-fix issues
4. Should-fix improvements
5. Polish opportunities
6. Experiments or open questions
7. Validation plan

For implementation tasks, convert the highest-impact must-fix or should-fix items into a small file-level plan before editing.
When benchmark calibration is used, include the ranked benchmark comparison and a clear recommendation to match, diverge, or experiment.

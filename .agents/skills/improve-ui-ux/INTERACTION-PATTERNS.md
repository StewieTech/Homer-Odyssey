# Interaction Patterns

Use these patterns to move from UX vocabulary to product changes.

## Direct Manipulation

Use when the learner is working with a phrase, correction, scenario, pronunciation clip, saved card, or chat turn.

Prefer:
- Inline editing
- Tap-to-replay
- Compare original vs corrected phrase in place
- Drag, reorder, save, or retry on the object itself

Watch for:
- Detached menus
- Hidden controls
- Editing flows that move the user away from the thing being edited

## Progressive Disclosure

Use when the screen has too many options or asks beginners to configure too early.

Prefer:
- One obvious primary path
- Advanced settings behind explicit expansion
- Contextual options after the user has taken the first step

Watch for:
- Feature tours before value
- Setup walls
- Multiple equal-weight calls to action

## Feedback Loop

Use when the user acts and needs to know what happened.

Prefer:
- Immediate local acknowledgement
- Specific result copy
- State change close to the action
- Progress that remains visible after the moment passes

Watch for:
- Silent taps
- Global toasts for local changes
- Spinners with no explanation

## Forgiving Input

Use when the user types, speaks, practices, pays, signs in, or submits language attempts.

Prefer:
- Input preservation
- Retry without reset
- Inline validation
- Undo or edit paths
- Error copy that explains recovery

Watch for:
- Dead ends
- Destructive actions without escape
- Corrections that sound punitive

## Recognition Over Recall

Use when the user must choose between scenarios, corrections, practice modes, or next steps.

Prefer:
- Visible examples
- Recent context near the choice
- Labels that forecast outcomes
- Side-by-side comparisons

Watch for:
- Hidden prior instructions
- Abstract labels
- Navigation that depends on memory

## Latency Masking

Use when AI, voice, network, billing, or auth may be slow.

Prefer:
- Immediate acknowledgement
- Streaming or staged results where possible
- Skeletons with layout stability
- Helpful waiting copy tied to the task

Watch for:
- Blank screens
- Jumping layouts
- Repeated loading indicators that erase context

## Low-Shame Correction

Use when the coach corrects grammar, pronunciation, fluency, or scenario choices.

Prefer:
- Specific correction
- Brief reason
- One-tap retry or replay
- Positive framing around progress
- User control over depth and timing

Watch for:
- Over-correction
- Public-feeling failure states
- Correction that interrupts the learning flow before the user is ready

## Confidence Loop

Use when the flow should help the learner feel ready for a real conversation.

Prefer:
- Attempt
- Feedback
- Retry
- Reuse in context
- Visible progress or readiness signal

Watch for:
- Feedback without practice
- Practice without progress
- Progress that feels arbitrary or hidden

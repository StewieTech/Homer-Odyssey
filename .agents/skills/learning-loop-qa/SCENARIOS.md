# Scenarios

V1 covers only activation-first flows.

## 1. Landing To First Practice

- Entry: `/`
- Goal: reach the first real practice moment with the assistant.
- Success: the learner sees a clear next action, makes an attempt, gets a useful the coach correction, and can repair.
- Common falloff: value-prop or action falloff.

## 2. Fast Mobile Activation

- Entry: `/` on mobile.
- Goal: complete the first learning loop with minimal attention.
- Success: the path stays short, readable, and fast.
- Common falloff: latency, layout, or setup friction.

## 3. Trust Before Attempt

- Entry: `/` or `/chat` when the learner hesitates about mic/login/cost/privacy.
- Goal: answer trust questions without forcing commitment too early.
- Success: the app explains recovery and keeps the next step obvious.
- Common falloff: permission or trust falloff.

## 4. Accessibility-First Entry

- Entry: `/` with keyboard-only or larger-text needs.
- Goal: start practice without hidden controls or unclear focus order.
- Success: the learner can find the main action, attempt, and recover.
- Common falloff: action or accessibility friction.

## Deferred V2 Scenarios

- Voice practice
- Returning-user resume
- Roleplay and place-practice flows
- Social-confidence or travel scenarios

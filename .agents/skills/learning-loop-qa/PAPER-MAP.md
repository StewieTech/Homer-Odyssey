# Paper Map

This is a translation layer, not a citation dump. Each paper becomes a QA principle.

| Paper | Distilled principle | V1 QA use |
|---|---|---|
| Cognitive Tutors: Lessons Learned | Adaptive tutoring should track mastery and choose the next best step | Score adaptive next-step quality and visible progress |
| AutoTutor | Conversational tutoring needs helpful moves, not just answers | Check whether the coach corrections are specific and actionable |
| Knowledge-Learning-Instruction Framework | Telemetry should prove learning, not just clicks | Define evidence for attempt, error type, repair, and success |
| Understanding and Using Context | Context should improve the next prompt without feeling creepy | Test whether prior context is useful and trustworthy |
| Speech and Language Processing | Speech and dialogue systems need latency and recognition discipline | Reserve for V2 voice QA; keep terminology ready now |
| Sentence-BERT | Meaning matters more than exact string match | Future-proof semantic grading and repair success checks |
| Case-Based Reasoning | Similar prior cases should inform current help | Shape memory and returning-user scenarios in V2 |
| Scripts, Plans, Goals, and Understanding | Structured scenarios help users know what kind of interaction they are in | Support scenario design and goal clarity |
| Game Feel | Responsiveness and reward feeling change engagement | Feed latency budgets, micro-feedback, and progress polish |

## Rule

- If a paper does not change a rubric item, scenario, telemetry rule, feedback rule, or product decision, do not keep it in this skill.

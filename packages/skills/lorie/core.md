# Governance review

Start only when the target names a launch, release, production, privacy, credential, billing, entitlement, legal, safety, external-side-effect, or user-trust boundary. Review the broad evidence portfolio, map the request to target-owned gates, identify what became more provable and what remains unprovable, state risk and confidence, and prepare a recommendation for the target's human decision owner.

Follow the [governance contract](./helpers/contract.md), [evidence harness](./helpers/evidence-harness.md), and [gate review contract](./helpers/gate-review.md). Current evidence must be source-linked, environment-aware, and reproducible enough for the decision. A no-op is valid when review adds no safe governance change and names the follow-up trigger.

Lorie is read-most and writes only requested governance artifacts. This skill does not implement, approve, merge, deploy, publish, change policy, mutate production, or execute the governed action.

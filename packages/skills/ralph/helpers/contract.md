# Worker contract

The worker packet names the accepted issue, scope and non-goals, resolved blockers, target repository, base branch and revision, branch identity, changed files, validation commands and results, residual risk, rollback, review target, and issue-closing linkage.

An issue is executable only when acceptance criteria are concrete, blockers are resolved, the slice is reversible and reviewable, and no planning, human, policy, or governance state remains. Provenance labels never override current lane state.

A validation or policy blocker stops the affected issue. Ralph may report unrelated pre-existing failures but never hides them or expands scope without a new accepted boundary.

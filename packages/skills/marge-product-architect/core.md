# Product architecture routing

Evaluate a route, module, workflow, or product behavior against a target-owned goal harness. Produce evidence-linked weighted dimension scores with confidence, apply the lowest relevant evidence cap, identify the smallest meaningful gap, and choose exactly one route: no-op, report only, bounded issue shaping, durable product direction, deeper planning, governance, human judgment, or explicitly gated execution handoff.

Read the [portable contract](./helpers/contract.md), [goal-harness contract](./helpers/goal-harness.md), [routing contract](./helpers/routing.md), and [human gates](./helpers/human-gates.md). Marge may synthesize and shape but does not implement. Never infer execution authority from an evaluation request; an execution handoff requires a target-defined mode, explicit human authorization, ready bounded work, and no unresolved governance gate.
